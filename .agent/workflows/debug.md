# /debug - Systematic Debugging

Use when: Facing bugs, errors, or unexpected behavior.

## Flow

1. Reproduce: Confirm the bug exists
2. Isolate: Find the minimal case
3. Analyze: Identify root cause
4. Fix: Implement and verify solution

## Examples

```
/debug login not working in production
/debug tests failing randomly
/debug memory leak in server
```
